# Сова (Sova) — AI Финансовый Помощник

**Дата:** 2026-04-08
**Статус:** Утверждён

## 1. Обзор продукта

**Сова** — AI-финансовый помощник с Telegram-ботом как основным интерфейсом и веб-дашбордом для глубокой аналитики. Продукт объединяет учёт финансов, управление инвестициями и персонализированные AI-рекомендации в одном месте.

### Ключевое отличие

Большинство финансовых приложений отлично логируют транзакции, но не говорят пользователю, что ДЕЛАТЬ с данными. Сова — это не трекер, а **советник**: анализирует, объясняет, рекомендует, торгует.

### Целевая аудитория

Русскоязычные пользователи с любым уровнем финансовой грамотности — от новичков до активных инвесторов. AI адаптирует стиль общения под уровень пользователя.

### Бизнес-модель

Freemium с pay-per-use для AI-функций:
- **Бесплатно:** учёт финансов, интеграции (ZenMoney, T-Bank), автокатегоризация, базовые запросы, уведомления
- **Платно (AI-баланс):** глубокая аналитика, инвестиционные рекомендации, персонализированные дайджесты, свободный AI-чат, торговые рекомендации

Оплата через Telegram Stars. Пользователь пополняет баланс (от 100₽), каждый AI-запрос списывает фиксированную сумму. Баланс бессрочный. Вывод средств возможен с уменьшением баланса.

### Компоненты

- **Основной сервис (FastAPI + aiogram 3)** — обрабатывает webhook бота, REST API для дашборда, CRUD операции, авторизация, биллинг. Синхронные операции.
- **AI Worker** — отдельный процесс, забирает задачи из Redis. Все LLM-вызовы (Claude API). Изолирован, т.к. запросы к LLM занимают 2-15 секунд.
- **Cron Worker** — периодические задачи: синхронизация ZenMoney (каждые 4 часа), обновление портфеля T-Bank (каждый час 7:00-23:00), парсинг новостей (каждые 2 часа), курсы валют, генерация дайджестов.
- **Trade Worker** — исполнение торговых операций через T-Bank gRPC API. Подтверждение → исполнение → проверка статуса → уведомление.

## 3. Telegram-бот

### Онбординг

1. `/start` → приветствие от Совы (3 предложения)
2. Выбор интеграций: "Подключить ZenMoney" / "Подключить T-Bank" / "Начать без интеграций"
3. Для каждой интеграции: OAuth-ссылка + выбор периода импорта ("За последний месяц" / "3 месяца" / "Всё время" / "Свой период")
4. Выбор уровня: "Я новичок" / "Уже веду бюджет" / "Инвестирую активно"
5. Готово — первый дайджест через 24 часа после синхронизации

### Быстрый ввод расходов

Бот понимает естественный язык:
- `кофе 350` → категория "Еда", 350₽
- `такси 600 работа` → категория "Транспорт", 600₽, тег "работа"
- Фото чека → OCR + автокатегоризация
- Неочевидная категория → бот спрашивает с inline-кнопками

### Меню (inline-кнопки)

```
/menu или "Меню"
├── 💰 Баланс          — текущие остатки по счетам
├── 📊 Сегодня          — траты за сегодня + сравнение с обычным днём
├── 📈 Портфель         — стоимость, P&L, топ-3 позиции
├── 🎯 Цели             — прогресс по активным целям
├── 🔔 Настройки        — уведомления, интеграции, уровень
├── 💎 Баланс AI        — остаток, пополнение, вывод
└── 📖 Помощь           — что умеет бот
```

### AI-команды (платные)

- "Что с моими финансами?" → анализ текущего состояния (5₽)
- "Как оптимизировать портфель?" → рекомендации по ребалансировке (8₽)
- "Стоит ли покупать {тикер}?" → анализ бумаги в контексте портфеля (10₽)
- "Куда уходят деньги?" → паттерны трат, аномалии (5₽)
- "Реально ли накопить 500к к декабрю?" → моделирование (5₽)
- "Дайджест" → принудительная генерация сводки (12₽)
- Свободный AI-чат (3₽ за сообщение)

Перед каждым платным запросом бот показывает стоимость и запрашивает подтверждение.

### Торговля

```
Пользователь: купить SBER 10 лотов
Бот: 📋 Заявка на покупку:
     SBER (Сбербанк) × 10 лотов
     Цена: ~2,850₽ за лот → ~28,500₽
     ⚠️ Не является финансовой рекомендацией
     [Подтвердить] [Отмена]
→ [Подтвердить]
Бот: ✅ Заявка исполнена: SBER × 10 по 2,847₽
     Комиссия: 8.54₽
```

Двойное подтверждение. Лимит на сделку без дополнительного подтверждения — 50,000₽ (настраивается). Превышение → дополнительный запрос.

| Приближение к лимиту | "Еда: 80% бюджета, осталось 4,200₽ на 8 дней" | Вкл |

| Персонализированная новость | "Газпром -4% — у тебя 15% портфеля в GAZP" | Вкл |

## 4. Веб-дашборд

### Авторизация

Вход через Telegram Login Widget — без паролей. Тот же аккаунт что в боте. Доступ из бота (кнопка "📊 Дашборд") или напрямую по URL.

### Страницы

**Главная (обзор):**
- Общий баланс по всем счетам
- Мини-график доходов/расходов за 30 дней
- Прогресс по целям (прогресс-бары)
- Стоимость портфеля + P&L за день
- Последние 5 транзакций
- Виджет "AI-инсайт дня" (бесплатный)

**Транзакции:**
- Таблица с фильтрами: период, категория, счёт, сумма, поиск
- Группировка: по дням / неделям / месяцам
- Массовое редактирование категорий
- Экспорт в CSV

**Аналитика:**
- Расходы по категориям (donut chart + тренд по месяцам)
- Доходы vs расходы (bar chart)
- Топ категорий с динамикой
- Сравнение месяц к месяцу, аномалии
- Кешфлоу heatmap по дням месяца

**Портфель:**
- Состав (pie chart по секторам, валютам, типам)
- Таблица позиций: тикер, кол-во, средняя/текущая цена, P&L, доля
- График стоимости портфеля во времени
- Дивидендный календарь
- История операций

**Цели:**
- Список с прогресс-барами
- Создание/редактирование: название, сумма, дедлайн, счёт
- Прогноз достижения на основе текущего темпа
- AI-рекомендация по ускорению (платно)

**Новости и рынок:**
- Персонализированная лента (AI-фильтрация под портфель)
- Курсы валют (USD, EUR, CNY)
- Индексы (MOEX, S&P 500)
- Каждая новость с пометкой: как влияет на портфель

**Настройки:**
- Интеграции: подключение/отключение, управление токенами
- Уведомления: триггеры и расписание
- Профиль: уровень, валюта отображения
- AI-баланс: история, пополнение, вывод
- Категории: управление пользовательскими категориями

### Адаптивность

Mobile-first (PWA). На телефоне дашборд работает как приложение. Основное мобильное взаимодействие — через бот, дашборд для глубокого анализа.

## 5. Интеграции

### ZenMoney

- **Подключение:** OAuth 2.0
- **Синхронизация:** diff-based API. Первый импорт — полный за выбранный период. Далее каждые 4 часа — дельта с последнего timestamp
- **Данные:** транзакции (сумма, дата, описание, категория, счёт), счета (баланс, валюта), категории
- **Маппинг категорий:** AI автоматически маппит при первом импорте. Пользователь может исправить, бот запоминает
- **Дедупликация:** если ручная трата совпадает с импортом из ZM (сумма + дата ± 1 день) — запрос подтверждения

### T-Bank Invest

- **Подключение:** API-токен (пользователь создаёт в приложении T-Bank)
- **Протокол:** gRPC (tinkoff-investments Python SDK)
- **Чтение (cron, каждый час 7:00-23:00):**
  - Портфель: позиции, средние цены, текущие цены
  - Операции: покупки, продажи, дивиденды, купоны, комиссии
  - Статистика: P&L по позициям и портфелю
- **Торговля (Trade Worker):**
  - Рыночные и лимитные заявки
  - Двойное подтверждение в боте
  - Статус: pending → executed/cancelled → уведомление
  - Настраиваемый лимит на сделку (по умолчанию 50,000₽)
- **Sandbox:** для тестирования торговых операций

### Парсинг новостей

**Источники:**
- RSS/API: РБК, Коммерсант, Investing.com
- MOEX API: индексы и инструменты
- ЦБ РФ API: курсы валют, ключевая ставка
- Telegram-каналы (опционально, Telethon)

**Процесс:**
1. Cron Worker собирает сырые новости каждые 2 часа → Redis
2. AI Worker обрабатывает пачку каждые 4 часа: фильтрация, извлечение фактов, оценка влияния
3. При генерации дайджеста/алерта — AI сопоставляет с портфелем пользователя

## 6. AI-система

### Модели

| Операция | Модель | Себестоимость | Цена юзера |
|----------|--------|---------------|------------|
| Категоризация | Haiku | ~0.3₽ | Бесплатно |
| Быстрый вопрос | Haiku | ~0.5₽ | Бесплатно |
| Анализ расходов | Sonnet | ~3₽ | 5₽ |
| Анализ портфеля | Sonnet | ~5₽ | 8₽ |
| Рекомендация по сделке | Sonnet | ~5₽ | 10₽ |
| Полный дайджест | Sonnet | ~8₽ | 12₽ |
| AI-чат (за сообщение) | Sonnet | ~2₽ | 3₽ |

### Контекст AI-запроса

Каждый запрос получает релевантный контекст:

```
System prompt (роль Совы, уровень пользователя, дисклеймер)
+
Финансовый профиль:
  - доходы/расходы за 3 мес (агрегированные по категориям)
  - текущие балансы
  - активные цели и прогресс
+
Портфель (если релевантно):
  - позиции, P&L, распределение
  - последние операции
+
Релевантные новости (из news_cache)
+
Запрос пользователя
```

Контекст адаптируется под уровень пользователя:
- **Новичок:** простой язык, объяснение терминов, осторожные советы
- **Продвинутый:** цифры, коэффициенты, сравнение с бенчмарками

### Дайджесты

**Ежедневный (утренний, если включён):**
- Баланс, изменение за день
- Курсы валют
- Портфель P&L, значимые движения
- 1 совет или факт

**Еженедельный (воскресенье 10:00):**
- Доходы vs расходы, сравнение с предыдущей неделей
- Топ-3 категории расходов с динамикой
- Прогресс по целям
- Портфель P&L, дивиденды
- 2-3 персонализированных инсайта
- Релевантные новости

**Формат:** структурированный текст с эмодзи в Telegram.

## 7. Модель данных

### Ключевые сущности

```sql
-- Пользователи
users (
    telegram_id BIGINT PRIMARY KEY,
    username VARCHAR,
    first_name VARCHAR,
    level VARCHAR DEFAULT 'beginner',  -- beginner | intermediate | advanced
    ai_balance DECIMAL(10,2) DEFAULT 0,
    notification_settings JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()

-- Интеграции
integrations (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    type VARCHAR NOT NULL,  -- zenmoney | tbank_invest

    sync_from_date DATE,
    last_synced_at TIMESTAMPTZ,
    status VARCHAR DEFAULT 'active'  -- active | error | disconnected

-- Счета
accounts (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    name VARCHAR NOT NULL,
    currency VARCHAR(3) DEFAULT 'RUB',
    balance DECIMAL(12,2),
    source VARCHAR NOT NULL,  -- zenmoney | manual
    external_id VARCHAR,
    UNIQUE(user_id, source, external_id)
)

-- Категории
categories (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,  -- NULL = системная
    name VARCHAR NOT NULL,
    icon VARCHAR,
    parent_id UUID REFERENCES categories,
    type VARCHAR NOT NULL  -- expense | income | transfer
)

-- Транзакции
transactions (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    account_id UUID REFERENCES accounts,
    category_id UUID REFERENCES categories,
    amount DECIMAL(12,2) NOT NULL,
    currency VARCHAR(3) DEFAULT 'RUB',
    date DATE NOT NULL,
    description TEXT,
    source VARCHAR NOT NULL,  -- zenmoney | tbank | manual | ocr
    external_id VARCHAR,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, source, external_id)
)

-- Позиции портфеля
portfolio_positions (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    ticker VARCHAR NOT NULL,
    figi VARCHAR,
    name VARCHAR,
    quantity INTEGER,
    avg_price DECIMAL(12,4),
    current_price DECIMAL(12,4),
    sector VARCHAR,
    asset_type VARCHAR,  -- stock | bond | etf | currency
    currency VARCHAR(3),
    updated_at TIMESTAMPTZ
)

-- Операции портфеля
portfolio_operations (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    ticker VARCHAR NOT NULL,
    operation_type VARCHAR NOT NULL,  -- buy | sell | dividend | coupon | commission
    quantity INTEGER,
    price DECIMAL(12,4),
    total DECIMAL(12,2),
    executed_at TIMESTAMPTZ
)

-- Цели
goals (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    name VARCHAR NOT NULL,
    target_amount DECIMAL(12,2) NOT NULL,
    current_amount DECIMAL(12,2) DEFAULT 0,
    deadline DATE,
    status VARCHAR DEFAULT 'active',  -- active | achieved | abandoned
    ai_recommendation TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
)

-- Лог AI-запросов
ai_usage_log (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    query_type VARCHAR NOT NULL,  -- insight | portfolio | trade_advice | digest | chat | withdrawal | topup
    cost DECIMAL(6,2),
    tokens_used INTEGER,
    created_at TIMESTAMPTZ DEFAULT NOW()
)

-- Кеш новостей
news_cache (
    id UUID PRIMARY KEY,
    source VARCHAR NOT NULL,
    title TEXT NOT NULL,
    url TEXT,
    published_at TIMESTAMPTZ,
    summary TEXT,
    affected_tickers TEXT[],
    sentiment VARCHAR  -- positive | negative | neutral
)

-- Торговые заявки
trade_orders (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users,
    ticker VARCHAR NOT NULL,
    direction VARCHAR NOT NULL,  -- buy | sell
    order_type VARCHAR DEFAULT 'market',  -- market | limit
    quantity INTEGER NOT NULL,
    price DECIMAL(12,4),
    status VARCHAR DEFAULT 'pending_confirmation',
    -- pending_confirmation | submitted | executed | cancelled | failed
    tbank_order_id VARCHAR,
    confirmed_at TIMESTAMPTZ,
    executed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
)
```

## 8. Безопасность

- **Токены интеграций:** шифрование AES-256, ключ в env-переменных
- **Торговля:** двойное подтверждение + настраиваемый лимит на сделку
- **AI-баланс:** нельзя уйти в минус; при выводе средств баланс уменьшается
- **Rate limiting:** 30 сообщений/минута на пользователя
- **Telegram webhook:** валидация подписи, HTTPS only
- **Дашборд auth:** Telegram Login Widget с проверкой hash по bot token
- **Дисклеймер:** при каждой торговой операции и инвестиционной рекомендации

- **Минимум:** 100₽
- **Баланс бессрочный** (не сгорает)
- **Автопополнение:** опционально, при балансе ниже X₽

### Вывод

- Доступная сумма = пополнения - списания - предыдущие выводы
- При выводе AI-баланс уменьшается
- Если баланс = 0 после вывода — платные функции недоступны
- Все операции (пополнение, списание, вывод) фиксируются в `ai_usage_log`

### Команды

```
/balance → текущий баланс + кнопки пополнения [+100₽] [+300₽] [+500₽]
/ai_history → история списаний и пополнений
/withdraw → вывод средств
```
