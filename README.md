# Product Development Pipeline

[🇬🇧 English version](README.en.md)

AI-конвейер полного цикла разработки продукта — от идеи до прода и обратно. Каждый этап выполняется специализированным AI-агентом, а решения на ключевых точках принимает человек.

![Архитектура пайплайна](product_pipeline.jpg)

## Как это работает

**Product Development Pipeline** — это фреймворк, в котором AI-агенты работают как команда разработки. Вы описываете идею продукта, а система:

1. Проводит исследование проблемы и рынка
2. Формирует продуктовое видение и бизнес-анализ
3. Проверяет юридические требования
4. Проектирует UX/UI
5. Создаёт архитектуру, пишет код, настраивает инфраструктуру
6. Тестирует и проверяет безопасность
7. Выпускает релиз
8. Запускает маркетинг
9. Собирает фидбек и аналитику

Каждый агент — это Claude Code субагент со своим системным промптом, правилами и навыками. Агенты передают артефакты друг другу по DAG-графу зависимостей.

### Принцип работы

- **Чат с проектом** — через дашборд, используя Anthropic API ключ
- **Запуск агентов** — через [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI (`claude --print`)
- **Артефакты** — каждый агент создаёт файлы в `projects/{project_id}/{phase}/{agent}/`
- **Состояние** — хранится в JSON (`orchestrator/state/{project_id}.json`)
- **Gate-точки** — человек принимает решения go/stop/pivot через дашборд

### Gate-точки (Human-in-the-Loop)

| Gate | Когда | Решения |
|------|-------|---------|
| **Gate 1: Строим?** | После Product Owner, перед Pipeline Architect | go / pivot / stop |
| **Gate 2: Архитектура** | После System Architect + UX/UI Designer, перед Tech Lead | go / revise / stop |
| **Gate 3: Go/No-go** | После QA + Security + DevOps, перед Release Manager | go / no-go / rollback |

## Быстрый старт

### Требования

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI установлен и авторизован
- Node.js 20+
- Python 3.10+
- Anthropic API ключ (для чата в дашборде)

### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/mogilevtsevdmitry/product-development-pipeline.git
cd product-development-pipeline

# 2. Настройте переменные окружения
cp orchestrator/.env.example orchestrator/.env
cp dashboard/.env.example dashboard/.env.local
# Отредактируйте файлы, добавьте ваши API-ключи

# 3. Установите зависимости дашборда
cd dashboard
npm install
cd ..

# 4. Запустите дашборд
cd dashboard
npm run dev
```

Дашборд будет доступен по адресу http://localhost:3344

### Создание первого проекта

1. Откройте дашборд в браузере
2. Нажмите «Новый проект»
3. Опишите идею продукта
4. Система создаст DAG-граф с блоками и агентами
5. Через чат настройте пайплайн под свой проект
6. Запустите выполнение

### Создание агентов

Агенты подхватываются автоматически — никакие конфиги вручную править не нужно.

**Способ 1 — через дашборд (рекомендуется):** на странице «Агенты» нажмите «Создать агента», заполните имя/фазу/роль. Дашборд создаст структуру папок, шаблоны `system-prompt.md` и `rules.md`, и зарегистрирует агента в `agents/agents-config.json`.

**Способ 2 — вручную:** просто создайте папку нужной структуры:

```
agents/{phase}/{agent-name}/
├── system-prompt.md   # Роль и задачи агента
├── rules.md           # Правила и ограничения
└── skills/            # Навыки (опционально)
    └── skill-name.md
```

При следующем запуске и дашборд, и оркестратор найдут нового агента автоматическим сканированием папки `agents/`.

Подробнее: [agents/README.md](agents/README.md)

## Структура проекта

```
product-development-pipeline/
├── orchestrator/          # Python: DAG executor, state machine
│   ├── engine.py          # Создание проектов, управление состоянием
│   ├── config.py          # Реестр агентов, фазы, gate-точки
│   ├── agent_runner.py    # Запуск агентов через Claude Code CLI
│   ├── gates.py           # Логика human gate-точек
│   ├── pipeline_builder.py # Динамическое построение DAG
│   └── .env.example       # Шаблон переменных окружения
│
├── dashboard/             # Next.js 15: веб-интерфейс
│   ├── src/
│   │   ├── app/           # Страницы и API routes
│   │   ├── components/    # React-компоненты (граф, чат, артефакты)
│   │   ├── lib/           # Типы, утилиты
│   │   └── mcp/           # MCP-серверы для чата
│   └── .env.example       # Шаблон переменных окружения
│
├── agents/                # Определения AI-агентов (ваши промпты)
│   └── README.md          # Формат и инструкция по созданию
│
├── mcp-servers/           # MCP-серверы для агентов
│   └── media-generator/   # Генерация медиа-контента
│
├── docs/                  # Спецификации и дизайн-документы
│   └── superpowers/specs/ # Архитектурные решения
│
└── projects/              # Артефакты проектов (gitignored, runtime)
```

## Режимы работы

| Режим | Описание |
|-------|----------|
| `auto` | Полная автоматизация, остановки только на gate-точках |
| `human_approval` | Подтверждение после каждого агента |

## Дашборд

Веб-интерфейс для управления пайплайном:

- **Главная** — список проектов
- **Проект** — интерактивный DAG-граф пайплайна, управление gate-точками
- **Чат** — общение с AI-ассистентом проекта (настройка блоков, агентов, зависимостей)
- **Артефакты** — просмотр результатов работы агентов
- **Дебаты** — дискуссия между агентами по спорным решениям

### API

| Endpoint | Описание |
|----------|----------|
| `GET /api/projects` | Список проектов |
| `GET /api/state/{id}` | Состояние проекта |
| `POST /api/state/{id}/gate` | Решение на gate-точке |
| `POST /api/chat` | Чат с проектом |
| `GET /api/agents` | Список агентов |
